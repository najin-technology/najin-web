# Naver Search Advisor (웹마스터 도구) 운영 SOP

## 배경

네이버는 **사이트맵 제출만으로는 색인 누락이 잦습니다**. 페이지별 "수집 요청"이 별도로 필요하며, 매주 모니터링하지 않으면 신규 글이 몇 주씩 색인되지 않을 수 있습니다.

이 문서는 매주 5~10분 안에 끝나는 정기 점검 절차입니다.

## 접근

- URL: https://searchadvisor.naver.com
- 로그인: 사이트 등록 시 사용한 네이버 계정
- 사이트: `https://www.najin-tech.com` 선택

---

## 매주 1회 점검 (월요일 권장)

### 1. 사이트 진단 (1분)

좌측 메뉴 → **사이트 관리 → 사이트 진단**

- 점수 확인 (90점 이상 유지)
- 경고 항목이 있으면 즉시 조치 (대부분 메타 누락, 이미지 alt 누락, 텍스트 부족 등)

### 2. 수집 요청 — 신규 글 (3분)

좌측 메뉴 → **요청 → 웹페이지 수집**

지난 1주일 동안 발행한 신규 글의 URL 을 **개별 입력**:
- `https://www.najin-tech.com/ko/posts/<slug>`
- `https://www.najin-tech.com/ko/notices/<id>`

⚠️ 한 계정당 일일 50개 한도. 평소엔 5~10개로 충분.

### 3. 인덱스 현황 (1분)

좌측 메뉴 → **검증 → 색인 현황**

- "전체 페이지 수" vs "색인된 페이지 수" 비교
- 색인률 80% 이상 유지가 목표
- 큰 격차가 보이면 → "색인 누락 이유" 항목 확인

### 4. 검색 노출·유입 (1분)

좌측 메뉴 → **분석 → 검색 노출**

- 지난 7일간 어떤 키워드로 노출됐나
- 클릭 수 추이 (전주 대비)
- "검색어가 곧 SEO 최적화 방향". 자주 노출되는 키워드를 콘텐츠 캘린더에 반영

### 5. 문제 페이지 (필요 시)

좌측 메뉴 → **검증 → 페이지 검증**

오류가 보고된 URL 만 우선 검토:
- Soft 404 (실제로는 200이지만 콘텐츠가 부실하다고 판단)
- 리다이렉트 체인 너무 길음
- 메타 description 중복
- 모바일 사용성 (대부분 잘 되어 있을 것)

## 월 1회 점검 (월말)

### 사이트맵 재제출

좌측 메뉴 → **요청 → 사이트맵 제출**

- `https://www.najin-tech.com/sitemap.xml` 상태 확인 (이미 등록됨)
- `https://www.najin-tech.com/sitemap-images.xml` 상태 확인
- 처리 결과: "성공" 인지 확인. 실패면 사이트맵 형식 점검.

### RSS 피드 등록

좌측 메뉴 → **요청 → RSS 제출**

- `https://www.najin-tech.com/feed.xml`
- 신규 글 발견 속도 ↑ (네이버는 RSS 적극 활용)

### 변경된 URL 일괄 요청

대량 업데이트(10개+ 글 수정)했다면:
- **요청 → 변경된 페이지 다수 제출** 활용

## 분기 1회 점검 (3개월마다)

### Robots.txt 검증

좌측 메뉴 → **검증 → robots.txt**

- 현재 정책에서 의도치 않게 차단되는 페이지 없는지
- `User-agent: Yeti` (네이버봇) 정책 정상 작동하는지

### 사이트맵 검증

좌측 메뉴 → **검증 → 사이트맵**

- 사이트맵 내 URL 수 vs 실제 색인 수 비교
- 누락 패턴이 보이면 (특정 라우트만 색인 안 됨) → 해당 페이지 수동 색인 요청

### 통계 분석

좌측 메뉴 → **분석 → 검색 노출/클릭**

월별 추이:
- 월 노출수 증가율
- 평균 노출 위치 (낮을수록 좋음, 1=1등)
- 핵심 키워드 순위 변화

## 트러블슈팅

### "색인 누락" 이유별 조치

| 이유 | 조치 |
|---|---|
| 페이지 발견 안 됨 | 수집 요청 + 사이트맵 재제출 |
| robots.txt 차단 | robots.ts 검토, 차단 의도 확인 |
| 메타 noindex | 페이지 메타 검증 (404/error 외엔 noindex 금지) |
| 콘텐츠 부족 | 본문 길이 ↑, 이미지 + 텍스트 보강 |
| 중복 콘텐츠 | canonical URL 명시, 다국어는 hreflang |
| 리다이렉트 체인 | 단일 hop으로 정리 |
| 모바일 사용성 | 페이지 검증 도구로 확인 |

### "요청 후 색인 안 되는 경우"

- **48시간 대기**: 네이버 색인은 즉각적이지 않음
- **재요청은 1주일 후**: 같은 URL 을 매일 재요청하면 스팸으로 인식
- **서버 응답 확인**: `curl -I https://www.najin-tech.com/ko/posts/<slug>` 로 200 응답 확인
- **페이지 자체 점검**: 모바일에서 직접 열어보고 콘텐츠가 풍부한지

### "검색결과에 안 보이는 경우"

- **색인 ≠ 노출**: 색인은 됐어도 순위가 낮으면 첫 페이지에 안 보임
- 도메인 권위가 낮은 신규 도메인은 정상. 백링크·콘텐츠 누적 후 점진적으로 개선
- "site:najin-tech.com 키워드" 로 검색해 인덱싱 자체는 됐는지 확인
